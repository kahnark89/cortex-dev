# AGENTS.md — cortex-dev

## What this repo is
A Node.js CLI tool that bootstraps and maintains Cortex — the unified AI collaboration governance layer combining the Genome Protocol (session coherence) with GroundLine (comprehension depth tracking).

## Genome (read before every session)
This repo uses the Cortex genome protocol on itself. Read these files at session start — in order:

1. `.genome/00_GENOTYPE.md` — architecture invariants and hard lines (protected)
2. `.genome/10_PHENOTYPE.md` — current sprint focus, open questions, next actions (update freely)
3. `.genome/20_EPIGENOME.md` — append-only decision log (append only, never edit)
4. `.genome/30_SELECTION.md` — architect heuristics and autonomous action policy (protected)
5. `.genome/40_SHADOW.md` — rejected paths and forbidden zones (append only, never edit)

After reading, update `10_PHENOTYPE.md` with any changes to focus, open questions, or next actions before ending the session.

## Repo structure
- `bin/cortex.js` — CLI entry point (all cortex commands)
- `src/` — TypeScript source (confidence, concepts, check, watcher, context, index)
- `src/__tests__/` — test suite (run via `npm test`, compiles to `.dist-test/`)
- `dist/` — compiled CommonJS output (generated by `npm run build`)
- `templates/.genome/` — the five genome template files copied by `cortex init`
- `templates/AGENTS.md` — the AGENTS.md template injected into user projects
- `INTEGRATION.md` — full cortex + animus-sdk integration guide

## Build and test
```bash
npm run build   # tsc → dist/ (no __tests__ compiled here)
npm test        # tsc -p tsconfig.test.json → .dist-test/, then node --test
```

## Core invariants (do not change without architect sign-off)
1. **Five genome files are a published interface.** Names, numbering, and roles cannot change without a major version bump.
2. **EPIGENOME and SHADOW are append-only.** The CLI must never modify or delete existing entries.
3. **SHADOW concepts block unconditionally.** Any matching concept produces an error in `cortex check`, regardless of score.
4. **Zero runtime dependencies.** No third-party packages in `src/` or `bin/`.
5. **`cortex context` is stdout-only JSON with no side effects.** Designed to be piped safely.
